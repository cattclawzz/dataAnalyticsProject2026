Comments are a method of conveying freeform information to the reader. They can be single line or multiline. Certain sequences of characters indicate to the interpreter or compiler what sections of the code to ignore within the syntax.

This chart represents comment tokens for single line comments.

The most common comment token is “//,” taking up 44.7% of the surveyed languages. It is most notable in some of the top languages including C/C++, JavaScript, and rust.

3.5% of the surveyed languages use other tokens not listed on the graph. Such as APL, which uses the lamp glyph (“⍝”); or true BASIC, which uses “!”.

Comment tokens are generally consistent across languages, sticking to just a few basic symbols. Using familiar symbols can be important when it comes to readability from the perspective of experienced programmers as they can carry their experience from one language to another.

This often differs in some esoteric languages because they are often intentionally designed to confuse the user. Comments are often visually distinct from the rest of the code through syntax highlighting, which is more apparent in block coding through separate graphics.
